1. 이 SOP의 목적
가상 인프라 로그 분석 연습: 교육용 가상 로그에서 'Link Down(연결 끊김)' 이벤트가 발견되었을 때, 핵심 데이터(발생 시간, 가상 장비명, 영향 범위, 후속 조치 여부)를 논리적으로 추출하고 정리하는 능력을 기르기 위함입니다.

직무 문서화 흐름 체화: 추출한 데이터를 바탕으로 교대 근무용 가상 메모나 장애 보고서 초안을 구성해 보며, 데이터센터 운영 부서의 협업 및 소통 방식을 연습하는 것을 목표로 합니다.

2. 절차 요약
Step 1 (식별): 샘플 로그를 검토하여 Link Down 이벤트가 기록된 정확한 발생 시간과 SAMPLE, DEMO, EDU 등의 접두어가 붙은 가상 장비명을 명확히 확인합니다.

Step 2 (추이 및 상태 분석): 해당 장비에서 동일한 이벤트가 반복적으로 발생하는지 로그 내부를 조사하고, Ticket opened나 Ticket escalated 같은 처리 상태 표현이 동반되었는지 점검합니다.

Step 3 (객관적 정리): 발생 원인을 임의로 추측하거나 단정 짓지 않고 로그에 기록된 사실만 정리하며, 판단이 모호한 사항은 Escalation(상급 인계) 필요 여부로 분류하여 표시합니다.

Step 4 (공유 준비): 정리된 결과물을 가상의 Handover Memo(인수인계 메모) 또는 Incident Report(장애 보고서) 초안에 바로 활용할 수 있도록 scannable하게 요약합니다.

3. 주의사항
실제 망 및 장비 접속 절대 금지: 어떠한 경우에도 실제 가동 중인 인프라 장비에 접속하거나 라이브 네트워크 진단 명령어를 실행해서는 안 되며, 오직 주어진 텍스트 로그 파일만 사용해야 합니다.

보안/대외비 데이터 생성 금지: 가상 시나리오를 채울 때 실제 IP 주소, 실제 장비 시리얼 번호, 실제 고유 티켓 번호 등 실물 정보처럼 오인될 수 있는 보안 민감 데이터를 기재하지 않습니다.

원인 단정 금지: 신호 단절 현상의 원인을 특정 하드웨어 불량이나 케이블 결함 등 단 하나의 이유로 섣불리 결론 내리지 않도록 유의합니다. 로그에 나온 사실만 근거로 삼아야 하며, AI가 원인을 단정할 경우 사람이 반드시 수정해야 합니다.

4. 마지막 확인항목
Notion 실습 페이지에 최종적으로 학습 내용을 기록하기 전, 아래 6가지 체크리스트를 통과했는지 반드시 점검합니다.

[ ] 가상 이벤트의 발생 시간이 포함되었는가?

[ ] 교육용 샘플 장비명을 정확히 명시했는가?

[ ] 로그 내에서 이벤트의 반복 여부를 확인하고 기록했는가?

[ ] 관련 Ticket 진행 상태 표현을 확인하여 매칭했는가?

[ ] 판단이 어려운 부분에 대해 Escalation 필요 여부를 표시했는가?

[ ] 실제 환경의 보안 데이터(실제 IP, 시리얼 등)가 완벽히 배제되었는가?

5. 학생이 이해하기 쉬운 한 문장 요약
"이 가상 지침서는 연결이 끊긴(Link Down) 샘플 로그를 마주했을 때, 원인을 섣불리 짐작하지 않고 오직 눈에 보이는 사실만 꼼꼼히 정리하여 다음 근무자에게 명확히 전달하는 연습을 하기 위한 지침입니다."



가상 실습 체크리스트
[1단계: 샘플 로그 분석 및 사실 기록]

□ [로그 확인] 샘플 로그에 기록된 Link Down 이벤트의 정확한 발생 시간을 확인하고 기록했는가?

□ [장비 식별] 이벤트가 발생한 가상 장비명에 SAMPLE, DEMO, EDU 접두어가 올바르게 포함되어 있는지 확인했는가?

□ [패턴 파악] 동일한 가상 장비에서 유사한 Link Down 이벤트가 반복해서 나타나는지 로그 내부를 조회했는가?

□ [티켓 연동] 로그 내에 Ticket opened 또는 Ticket escalated라는 가상 처리 상태 표현이 동반되었는지 확인했는가?

[2단계: 객관적 서술 및 검토]

□ [원인 단정 방지] 물리적 원인을 임의로 확정 짓지 않고, 오직 로그에 나타난 관찰 내용(사실)만 객관적으로 기술했는가?

□ [리스크 분류] 혼자 판단하기 모호하거나 추가 분석이 필요한 지점에 대해 Escalation 필요 여부를 명시했는가?

[3단계: 가상 문서화 및 보안 점검]

□ [인계 준비] 확인된 내용을 가상의 Handover Memo 또는 Incident Report 초안에 바로 활용할 수 있도록 짧고 scannable하게 정리했는가?

□ [보안 필터링] 실제 운영 환경의 IP 주소, 실물 장비 시리얼 번호, 실제 고유 티켓 번호 등 실제 정보가 철저히 배제되었는지 다시 한번 확인했는가?
